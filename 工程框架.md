# 工程框架

## 1. 分层结构

当前工程按照“板级驱动层 -> 中间层 -> 应用层”的方式组织，核心模块如下：

1. 串口板级驱动层
   - `uart/src/debug_uart/bsp_debug_uart.h`
   - `uart/src/debug_uart/bsp_debug_uart.c`
   - 负责封装 `g_uart2`、`g_uart7`。
2. 串口中间层
   - `uart/src/debug_uart/mid_uart.h`
   - `uart/src/debug_uart/mid_uart.c`
   - 负责统一提供字节流、字符串、16 进制命令等接口。
3. 协议层
   - `uart/src/debug_uart/glove_frame.h`
   - `uart/src/debug_uart/glove_frame.c`
   - 负责解析 STM32 发来的固定 31 字节特征帧。
4. 算法层
   - `uart/src/gesture/gesture_model.h`
   - 负责 SVM 模型参数和分类公式。
5. 手势业务层
   - `uart/src/gesture/gesture_service.h`
   - `uart/src/gesture/gesture_service.c`
   - 负责 `100ms` 样本更新、分类、命令映射、命令过滤。
6. 定时器板级驱动层
   - `uart/src/timer/bsp_dispatch_timer.h`
   - `uart/src/timer/bsp_dispatch_timer.c`
   - 负责封装 `g_timer_agt0` 和 `timer0_callback()`。
7. 定时器中间层
   - `uart/src/timer/mid_dispatch_timer.h`
   - `uart/src/timer/mid_dispatch_timer.c`
   - 负责把 `10ms` Tick 聚合成 `200ms` 发送标志。
8. 应用层
   - `uart/src/hal_entry.c`
   - 负责把各层串起来，并最终通过 `uart7` 发送原始命令字节。

## 2. 当前时序

1. `uart2` 持续接收 STM32 每 `20ms` 发来的一帧特征数据。
2. 业务层每 `100ms` 更新一次分类样本。
3. `AGT0` 每 `10ms` 进入一次中断。
4. 定时器中间层累计 `20` 次中断后，置位一次 `200ms` 发送标志。
5. 应用层拿到标志后执行一次分类。
6. 如果分类结果满足过滤条件，则通过 `uart7` 发送一个原始 16 进制命令字节。

## 3. 当前命令发送规则

1. 只在 `200ms` 周期点发送。
2. 只发原始 16 进制命令字节，不发任何文本。
3. `0x02` 不发送。
4. `0x00` 不发送。
5. 其他命令必须连续 `2` 次分类结果一致才发送。
