# 工程框架

## 1. 分层结构

当前 `uart` 工程按“板级驱动层 -> 中间层 -> 业务层 -> 应用层”组织，新增了手势数据采集与离线训练闭环。

### 1.1 板级驱动层

1. 串口驱动层
   - `uart/src/debug_uart/bsp_debug_uart.h`
   - `uart/src/debug_uart/bsp_debug_uart.c`
   - 负责封装 `g_uart2`、`g_uart7` 的打开、收发、回调与阻塞等待
2. 定时器驱动层
   - `uart/src/timer/bsp_dispatch_timer.h`
   - `uart/src/timer/bsp_dispatch_timer.c`
   - 负责封装 `g_timer_agt0` 和 `timer0_callback`
3. 按键驱动层
   - `uart/src/key/bsp_key_irq.h`
   - `uart/src/key/bsp_key_irq.c`
   - 负责配置 SW2 外部中断，并锁存“按下一次”的事件
4. LED 驱动层
   - `uart/src/led/bsp_led.h`
   - `uart/src/led/bsp_led.c`
   - 负责 LED 初始化和亮灭控制

### 1.2 中间层

1. UART 中间层
   - `uart/src/debug_uart/mid_uart.h`
   - `uart/src/debug_uart/mid_uart.c`
   - 对上统一提供字节流、字符串、浮点文本等接口
2. 协议解析层
   - `uart/src/debug_uart/glove_frame.h`
   - `uart/src/debug_uart/glove_frame.c`
   - 负责把 STM32 发来的固定长度二进制帧解析为 `glove_frame_t`
3. 调度定时中间层
   - `uart/src/timer/mid_dispatch_timer.h`
   - `uart/src/timer/mid_dispatch_timer.c`
   - 负责把 10ms Tick 聚合成 200ms 业务节拍，并支持模式切换时清空节拍状态
4. 按键事件中间层
   - `uart/src/key/mid_key_event.h`
   - `uart/src/key/mid_key_event.c`
   - 对应用层暴露“按键事件”语义，隔离 ICU 中断实现细节

### 1.3 业务层

1. 模型适配层
   - `uart/src/gesture/gesture_model.h`
   - 直接包含 `uart/tools/generated/gesture_model.h`
   - 固件不再手写 SVM 权重数组
2. 手势识别业务层
   - `uart/src/gesture/gesture_service.h`
   - `uart/src/gesture/gesture_service.c`
   - 负责 100ms 样本更新、200ms 分类、命令映射、命令过滤
3. 手势采集导出业务层
   - `uart/src/gesture/gesture_capture_service.h`
   - `uart/src/gesture/gesture_capture_service.c`
   - 负责把有效帧转换成上位机可直接保存的 10 列 CSV 文本

### 1.4 应用层

1. 应用入口
   - `uart/src/hal_entry.c`
   - 负责模式切换、主循环编排、识别链路与采集链路的互斥调度

### 1.5 上位机工具层

1. 数据采集工具
   - `uart/tools/collect_data.py`
   - 负责读取开发板 `uart7` 输出的 10 列 CSV，并在 PC 本地保存带标签数据
2. 模型训练导出工具
   - `uart/tools/train_svm.py`
   - 负责读取 CSV、训练线性 SVM、导出 `uart/tools/generated/gesture_model.h`
3. 数据与模型目录
   - `uart/tools/data/gesture_data.csv`
   - `uart/tools/generated/gesture_model.h`

## 2. 运行模式

`uart` 固件现在有两种互斥模式：

1. 识别模式
   - 默认上电进入
   - `uart2` 接收 STM32 特征帧
   - `gesture_service` 负责采样、分类、命令确认
   - `uart7` 只发送 1 字节命令
   - LED 熄灭，识别到有效命令时闪烁
2. 采集模式
   - 按一次 SW2 切换进入
   - `uart2` 仍然接收 STM32 特征帧
   - `gesture_capture_service` 把有效帧转成 10 列 CSV
   - `uart7` 连续输出特征文本，不再发送命令
   - LED 常亮
   - 再按一次 SW2 退出，恢复识别模式

## 3. 数据链路

### 3.1 识别链路

1. STM32 每帧通过 `uart2` 发送固定长度二进制特征帧
2. `GloveFrame_Receive()` 完成同步、CRC 校验和字段解析
3. `GestureService_ProcessFrame()` 维护 100ms 样本缓存
4. `MID_DispatchTimer_TryConsumeDispatchFlag()` 每 200ms 提供一次分类节拍
5. `GestureService_RunClassifyCycle()` 进行 SVM 分类与命令过滤
6. 满足发送条件后，`hal_entry.c` 通过 `uart7` 发出 1 字节命令

### 3.2 采集链路

1. STM32 每帧通过 `uart2` 发送固定长度二进制特征帧
2. `GloveFrame_Receive()` 解析出 `glove_frame_t`
3. `GestureCaptureService_ProcessFrame()` 仅导出 `status` 含 `DATA_VALID` 的帧
4. `uart7` 连续输出：
   - `f0x,f0y,f1x,f1y,f2x,f2y,f3x,f3y,roll,pitch`
5. `uart/tools/collect_data.py` 从串口读取这些行
6. 上位机按标签保存到 `uart/tools/data/gesture_data.csv`

## 4. 模型更新流程

现在模型更新已经闭环，后续不需要再去 `uart/src` 里手抄权重。

### 4.1 采集数据

1. 烧录 `uart` 工程
2. 将 PC 连接到开发板 `uart7`
3. 按一次 SW2 进入采集模式
4. 执行：

```bash
cd uart/tools
python collect_data.py --port COM7
```

5. 按工具提示采集 9 类手势数据

### 4.2 训练模型

执行：

```bash
cd uart/tools
python train_svm.py
```

输出文件：

`uart/tools/generated/gesture_model.h`

### 4.3 固件使用新模型

`uart/src/gesture/gesture_model.h` 已经固定包含：

`uart/tools/generated/gesture_model.h`

所以后续更新模型时，只需要：

1. 重新采集 `CSV`
2. 重新运行 `train_svm.py`
3. 重新编译 `uart` 工程

核心结论：只替换权重文件，就可以让工程运行新的 SVM 模型。

## 5. 当前关键约束

1. 当前协议层固定输出 10 维特征
   - `f0x,f0y,f1x,f1y,f2x,f2y,f3x,f3y,roll,pitch`
2. 当前业务层命令映射固定使用 9 类标签
   - `fist/open/one/two/rock/up/down/left/right`
3. `train_svm.py` 会检查标签是否完整
   - 如果不是 0~8 共 9 类，会直接拒绝导出
4. 识别模式和采集模式互斥
   - 防止 `uart7` 上混发命令字节和 CSV 文本
