# STM32 手套端工程大纲

## 1. 当前工程定位

该工程对应迁移方案中的 STM32 手套前端，当前职责是：

1. 继续完成 MPU6050 与 JY901S 采集
2. 复用原有 10 维特征定义
3. 通过固定长度二进制帧把特征发送给 RA6M5

当前不再以“本地分类后发送 1 字节命令”为主链路。

## 2. 关键目录

### 2.1 应用入口

- `shoutao_you/Core/Src/main.c`

职责：

1. 初始化外设
2. 轮询多路 MPU6050
3. 按固定节拍触发特征帧发送

### 2.2 串口发送层

- `shoutao_you/MDK-ARM/HARDWARE/trans.h`
- `shoutao_you/MDK-ARM/HARDWARE/trans.c`

职责：

1. 组织 10 维特征
2. 构造固定帧格式
3. 计算 CRC16
4. 通过 `huart6` 发送给 RA6M5

### 2.3 中断与节拍层

- `shoutao_you/MDK-ARM/HARDWARE/my_nvic.c`

职责：

1. TIM6 每 200ms 置位 `trans_flag`
2. 主循环检测到 `trans_flag` 后发送一帧特征数据
3. 迁移模式下禁止按键往 `huart6` 直接塞单字节命令

### 2.4 调试串口重定向

- `shoutao_you/Core/Src/usart.c`

职责：

1. 迁移模式下把 `printf` 从 `huart6` 挪到 `UART4`
2. 避免文本调试信息污染发给 RA6M5 的二进制特征帧

## 3. 当前发送协议

帧结构如下：

```c
| sof1 | sof2 | ver | seq | tick_ms | feature[10] | status | crc16 |
```

字段说明：

1. `sof1` = `0xAA`
2. `sof2` = `0x55`
3. `ver` = 协议版本号，当前为 `0x01`
4. `seq` = 帧序号
5. `tick_ms` = `HAL_GetTick()` 时间戳
6. `feature[10]` = 10 维特征，按 `float * 100` 转成 `int16_t`
7. `status` = 状态字
8. `crc16` = CRC16-CCITT-FALSE

总长度固定为 31 字节。

## 4. 特征顺序

发送特征顺序与迁移方案保持一致：

1. `finger1_x` -> `mpu_data[0].filter_angle_x`
2. `finger1_y` -> `mpu_data[0].filter_angle_y`
3. `finger2_x` -> `mpu_data[1].filter_angle_x`
4. `finger2_y` -> `mpu_data[1].filter_angle_y`
5. `finger3_x` -> `mpu_data[2].filter_angle_x`
6. `finger3_y` -> `mpu_data[2].filter_angle_y`
7. `finger4_x` -> `mpu_data[3].filter_angle_x`
8. `finger4_y` -> `mpu_data[3].filter_angle_y`
9. `roll` -> `jy901s_data.angle_roll`
10. `pitch` -> `jy901s_data.angle_pitch`

## 5. 当前迁移模式开关

在 `trans.h` 中定义：

```c
#define GLOVE_MIGRATION_PROTOCOL_ENABLE 1U
```

含义：

1. `1`：启用迁移模式，发送固定特征帧
2. `0`：恢复旧版 1 字节命令发送逻辑

## 6. 当前注意事项

1. `huart6` 当前专门用于发送给 RA6M5 的二进制帧
2. 迁移模式下不要再往 `huart6` 输出文本调试信息
3. 如果 RA6M5 端收包异常，优先检查：
   1. 帧头 `0xAA 0x55`
   2. 31 字节总长度
   3. `feature[10]` 是否按小端 `int16_t` 解析
   4. CRC16 算法是否与 STM32 端一致
